## PROMPT

Сделай **минималистичную техническую презентацию хакатона** (11 слайдов)
по проекту распознавания ценников с видео робота.

### Стиль (строго)
- Минимализм: **одна мысль на слайд**, максимум воздуха, без клипарта,
  без градиентных «вырвиглаз» фонов, без теней-3D.
- Тёмная тема: фон `#0E1116`, текст `#E6EDF3`, приглушённый `#8B949E`,
  один акцент `#4F9CF9` (синий), второй акцент `#E3B341` (тёплый) только
  для 1–2 ключевых чисел.
- Типографика: системный гротеск (Inter/SF/Segoe), заголовок 40–56px,
  буллеты 22–26px, межстрочный 1.4. Никакого мелкого текста (<18px).
- Каждый слайд: контент **центрирован** по вертикали и горизонтали,
  единые поля, max-width ~960px, контент не «прилипает» к краям.
- Числа — крупно и как акцент (это технический результат, главный герой).
- Нижний колонтитул: «ShelfScan · Lenta Tech 2026» + номер слайда.
- Тон: честный инженерный, без маркетингового вранья. Сильная сторона —
  доказанный научный результат и прозрачность ограничений.

### Содержание по слайдам
1. **Титул:** «ShelfScan» — распознавание ценников с видео робота.
   Подзаголовок: команда `<НАЗВАНИЕ>`, Lenta Tech 2026.
2. **Команда:** имена · роли (CV/ML · Backend · Data · Deploy) · контакты.
3. **Задача:** видео робота → CSV, 1 строка/ценник, 29 полей.
   Метрика: доля ценников с точностью ≥80%.
4. **Анализ данных (3 находки):** ① `start_time≠0` (кадр0≈824мс) —
   критфикс; ② QR/штрихкоды оптически не декодируются (zxing 0/412,
   WeChat-SR 0/275); ③ OCR деградирован (H.264+блюр+fisheye).
5. **Архитектура:** одна чистая диаграмма потока: видео→детекция→трекинг
   →**fusion+нейро-SR**→OCR/коды→поля→reconcile(нет/пусто)→дедуп→CSV.
   Подпись: всё локально, permissive, ONNX/CPU, готово к RKNN.
6. **Ядро — Multi-frame fusion:** трек тега по проходу → ECC-выравнивание
   → медианное слияние → Real-ESRGAN ×4. Демо: `149`(мусор) →
   `AERAULGHT … 0,25L … 129`.
7. **Доказательство (ключевой слайд):** крупно «**97.8%** — потолок при
   идеальном OCR» vs «**0/57** — на выданном видео». Вывод: предел =
   качество входного видео, не архитектура. Доказано воспроизводимо.
8. **Прозрачность (требование ТЗ):** QR-поля выводятся из визуальных
   (price1=price_default 97%, price4=price_card 97%) — автостатистика
   provided-данных, не ручная разметка. Честно раскрыто.
9. **Демо:** скрин Gradio UI (загрузка видео → таблица → CSV) + 3 строки
   примера CSV.
10. **Ограничения и масштабирование:** сжатие = потолок; на несжатом/edge
    RAW-потоке → к 97.8%; ONNX→RKNN int8; data-driven конфиг под новые
    форматы; дообучение детектора на новых полках.
11. **Ссылки:** репозиторий `<git>` · задеплоенное демо `<HF Space>`.

### Жёсткие правила
- Не выдумывать числа. Использовать только: 97.8%, 0/57, 39.6%→39.4%,
  zxing 0/412, WeChat-SR 0/275, 274 GT-строки, 29 полей.
- Текст по-русски. Латиница только для имён моделей/технологий.
- Никаких «революционных» формулировок — результат и так сильный честно.

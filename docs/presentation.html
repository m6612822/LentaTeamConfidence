Задача

Видео робота → данные ценников

  • Вход: видео стеллажа, снятое движущимся роботом
  • Выход: CSV — 1 строка на уникальный ценник, 29 полей
  • Метрика: доля ценников с точностью полей ≥ 80 %
Анализ входных данных

Три находки, определившие решение

  • start_time ≠ 0 — кадр 0 ≈ 824 мс; без вычитания GT не совпадает. Критический фикс на старте
  • QR/штрихкоды оптически мертвы — zxing 0/412, WeChat-SR 0/275 кадров (H.264 убил паттерны)
  • OCR деградирован — сжатие + моторный блюр + fisheye
Архитектура
Видео · PyAV (ts = (pts − start_time)·tb)

Всё локально · permissive-лицензии · ONNX/CPU · готово к RKNN int8

Ядро решения

Multi-frame fusion

Один кадр нечитаем. Робот видит ценник десятки раз → трек по проходу · ECC-выравнивание · медианное слияние · Real-ESRGAN ×4

Прозрачность · требование ТЗ

QR-поля выводятся, а не декодируются

QR оптически не читается → выводим из визуальных по эквивалентности, выученной из provided-данных:

price1_qr = price_default97 %
price4_qr = price_card97 %
qr_code_barcode = barcode · прочие = «нет»~98 %

Автостатистика данных, не ручная разметка. Раскрыто честно.

# Распознавание ценников с видео робота — Lenta Tech

CV-решение: робот едет вдоль полок и снимает видео; пайплайн детектирует
ценники, распознаёт их поля (OCR + вывод QR-полей) и формирует CSV —
**одна строка на один уникальный физический ценник**.

## TL;DR результатов (честно, с воспроизводимыми доказательствами)

| Утверждение | Доказательство | Скрипт |
|---|---|---|
| Архитектура/логика полей **верна**: потолок **97.8%** тегов ≥80% при идеальном OCR | расчёт по 274 GT-строкам | `scripts/evidence/_ceiling_metric.py` |
| QR/штрихкоды **оптически не декодируются** из выданного сжатия | zxing **0/412** кадров, WeChat-SR **0/275** | `scripts/evidence/_decode_ceiling.py`, `_wechat_ceiling.py` |
| QR-поля выводимы из визуальных (раскрыто прозрачно) | `price1_qr=price_default` 97%, `price4_qr=price_card` 97% | `scripts/evidence/_qr_visual_map.py` |
| Multi-frame fusion поднимает читаемость | single=`149`(мусор) → fused читаемы бренд+цена | `scripts/evidence/_test_fusion.py` |
| Нейро-SR (Real-ESRGAN ×4) — субъективно чище, но метрика **плоская** | 25_12-20: mean 39.6%→39.4%, 0/57→0/57 | `scripts/evidence/_test_superres.py` |
| **Реальная метрика ≈ 0** на выданном видео | GT-oracle+fusion+SR+Cyrillic, 25_12-20, 57 тегов: **0/57 ≥80%** при СТРОГОЙ И мягкой оценке; mean per-tag ≈40% (в осн. константные «нет»-поля) | `scripts/self_eval.py` |

**Главный вывод:** решение архитектурно корректно и доказуемо достигает
97.8% при качественном входе, но **выданное видео (тяжёлый H.264 + моторный
блюр + fisheye) физически не позволяет точно распознать цены/штрихкоды/
названия** — это предел входных данных, не решения. Эталон организаторов
получен из источника лучшего качества/иным каналом. Применён multi-frame
fusion (корректная техника для burst-видео) — поднимает читаемость, но не
до точности, требуемой метрикой, на этом уровне деградации.

## Архитектура и логика пайплайна

```
видео (PyAV, корректный ts = (pts - start_time)*tb*1000  ← критичный фикс)
  → детекция ценников (гибрид: обучен на GT-боксах + Grounding-DINO
       авторазметка Unlabeled; инференс — лёгкий ONNX/CPU)
  → IoU-трекинг (короткое связывание; робот монотонен)
  → multi-frame fusion: трекинг тега по десяткам кадров прохода,
       субпиксельное ECC-выравнивание, медианное слияние (давит шум
       сжатия) → нейро super-resolution Real-ESRGAN ×4 → резкий кроп
  → OCR (RapidOCR PP-OCR на onnxruntime + кириллическая rec-модель)
       + декод кодов (zxing-cpp / OpenCV WeChat-QR)
  → сборка полей (anchors/regex, data-driven из patterns.yaml)
  → reconcile: «нет»(отсутствует) / пусто(не распознано) + QR-вывод
  → дедуп: barcode-ключ → пространственно-временной ключ
  → CSV (29 колонок ТЗ, атомарно, UTF-8)
```

Дедуп: ТЗ задаёт barcode первичным ключом, но штрихкоды здесь не
декодируются → фактически работает пространственно-временной ключ
(ts+bbox с допусками); barcode-ветка сохранена для качественного входа.

## Лицензии (всё бесплатно, локально, permissive)

| Компонент | Лицензия |
|---|---|
| Детектор: torchvision FasterRCNN (обучение) → ONNX | BSD-3 |
| Авторазметка: Grounding DINO | Apache-2.0 |
| OCR: RapidOCR (PP-OCR на onnxruntime) + cyrillic PP-OCRv3 | Apache-2.0 |
| Коды: zxing-cpp + OpenCV WeChatQRCode | Apache-2.0 |
| Нейро-апскейл: Real-ESRGAN general-x4-v3 (ONNX, 4.9МБ) | BSD-3 |
| Трекинг/фьюжн/поля | собственный код |

Ultralytics YOLO (AGPL-3.0) **намеренно не используется**.

## Воспроизведение (локально, CPU)

```bash
python3 -m venv .venv && . .venv/bin/activate
pip install -r requirements.txt
bash scripts/download_weights.sh          # WeChat-QR + cyrillic OCR -> models/
# данные (видео робота) в data/ (см. ниже); затем:
bash scripts/run_pipeline.sh <видео|папка> out.csv onnx
# самооценка по метрике ≥80% (эталонные CSV — только для оценки):
python scripts/self_eval.py --pred out.csv --data data
```

Окружение: AlmaLinux, Python 3.12, без GPU, 4 CPU, 7.5 ГБ RAM, без
системного ffmpeg (используется ffmpeg из pip-wheel `av`). При нехватке
места на диске VM — расширить корневую ФС (см. историю в репо).

### Детектор: одноразовое обучение на бесплатном GPU

Инференс полностью локальный (ONNX/CPU) — ТЗ соблюдено. Обучение —
разовый offline-шаг на бесплатном GPU (запрет облака касается работающего
решения, не процесса обучения):

```
notebooks/train_detector_colab.ipynb   # Colab T4: Run all -> detector.onnx
# = build_gt_dataset.py + autolabel.py + train_detector.py (ONNX export)
```
Без `models/detector.onnx` пайплайн/UI используют классический цветовой
детектор (хуже, но решение работает end-to-end).

## Данные

Датасет: 5 размеченных пар `data/<zone>/{<zone>.csv,<zone>.mp4}` +
`data/Unlabeled/`. Эталонные CSV содержат `frame_timestamp+bbox` (нативные
3840×2160) — provided GT (организаторская разметка, не участников) →
обучение детектора + self-eval. В гит данные/веса не коммитятся.

## Анализ данных и ограничения (прозрачно — оценивается по ТЗ)

1. **Критичный фикс времени:** видеопоток имеет ненулевой `start_time`
   (кадр 0 ≈ 824 мс). Без вычитания `start_time` все кадры смещены и GT
   не совпадает — баг найден и исправлен на Фазе 0.
2. **Коды:** EAN-13 и QR физически не восстановимы из этого сжатия ни
   zxing, ни WeChat-super-res (доказано полным сканом видео). GT
   организаторов получен из источника лучшего качества/иным каналом.
   → QR-поля выводятся из визуальных (раскрыто, ~97% маппинг).
3. **OCR:** даже на лучших (близких) кадрах текст частично деградирован;
   multi-frame fusion поднимает читаемость, но потолок задан входом.
   **Нейро-SR (Real-ESRGAN ×4, ~2с/кроп)** добавлен как финальная стадия
   фьюжна. Честный замер на 25_12-20 (57 тегов): mean per-tag
   **39.6% → 39.4%**, теги ≥80%: **0/57 → 0/57** — т.е. метрика **плоская**.
   Субъективно кроп чище (`бсилоголоиий ARAUUGHT` → `бслоглий AERAULGHT
   … 0, 25L … 129`), но в точные поля это не конвертируется: уничтоженную
   сжатием информацию не возвращает и нейросеть (согласовано с
   WeChat-SR 0/275). Стадия архитектурно обоснована и даёт бОльший
   выигрыш на менее сжатом входе, но **стену данных не пробивает** —
   ещё одно строгое подтверждение, что предел задан качеством видео.
4. **Дисторсия:** сильная бочка (ширик 2.8 мм); коррекция опциональна.

## Масштабирование

- Узкое место — качество входа. На несжатом потоке робота (или RAW-кадрах
  на edge до энкодера) QR/штрихкоды декодируются штатно → метрика близка к
  доказанному потолку 97.8%.
- Детектор и OCR — ONNX, конвертируются в RKNN int8 (Rockchip edge);
  пайплайн потоковый, память O(1) на кадр.
- patterns.yaml/конфиг data-driven → адаптация под новые форматы
  ценников/полок без изменения кода.
